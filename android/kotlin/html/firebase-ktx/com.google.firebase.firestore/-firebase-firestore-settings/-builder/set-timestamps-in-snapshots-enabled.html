setTimestampsInSnapshotsEnabled

fun setTimestampsInSnapshotsEnabled(value: Boolean): FirebaseFirestoreSettings.Builder!

Specifies whether to use Timestamps for timestamp fields in DocumentSnapshots. This is now enabled by default and should not be disabled.

Previously, Firestore returned timestamp fields as java.util.Date but only supports millisecond precision, which leads to truncation and causes unexpected behavior when using a timestamp from a snapshot as a part of a subsequent query.

So now Firestore returns Timestamp values instead of java.util.Date, avoiding this kind of problem.

To opt into the old behavior of returning Dates, you can temporarily set FirebaseFirestoreSettings#areTimestampsInSnapshotsEnabled to false.
